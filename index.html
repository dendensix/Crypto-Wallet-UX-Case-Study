The Future for a Decentralized Wallet

In this case study, I'll be offering my take on the future for a Decentralized Crypto Wallet, in comparison with the most popular one to this day: MetaMask. My process was guided by qualitative user research, MetaMask's current design, and my own intuition as a UX designer.

Introduction

As a designer who's highly interested in the world of crypto and web3, I soon came to the realization that there's a lot of room for improvement in the area of User Experience. Through design, I hope to help build the future of DeFi, in a way that facilitates everyone, including your grandma, to use SaaS applications without confussion or irritation.
However, the current stage of MetaMask, the #1 decentralized crypto wallet in the world, with over 30 million active users, has a lot to catch up to.
I approached my redesign in two steps:
- Core Experience
- Visual Interface

1. Core Experience

One of the biggest problems with MetaMask's current infrastructure is the way information and actions are nested and organized. Most of the critical information is hidden under buttons and drop-downs that make browsing through your assets insufferable.
While it makes sense to consolidate information on a mobile application or website due to the more restricted real estate area, an expanded desktop web app has more room for improved functionality and usability.
MetaMask has two main window views:
- The minimized pop-up window
- The full-width expanded window
This is what MetaMask's expanded and minimized windows look like.
However, there's absolutely no difference between the two. The full-width expanded window has a lot of available white space that would allow for great improvements in usability.
Here's my take on how to optimize the core experience. Starting with...

Networks & Tokens

Ethereum Mainnet is displayed as the default network, along with the tokens you own within that network. Say you want to see any Binance tokens you own. You'd first have to switch networks by clicking on Ethereum Mainnet, pulling a drop-down menu, and selecting the right network â€” in this case: Binance.
Currently, there is no way to see all of your tokens at once. Every single one is hidden under each network, which you have to browse individually, every single time. Why though?
Wouldn't it be helpful to display all your tokens within a specific account in one place, neatly categorized in "network folders"?
That's exactly what I did. I chose to display all the tokens the user owns in a specific account in one place, each categorized under its corresponding network. These network folders can be folded or minimized to save space, and stay organized.
Doesn't it look tidy? All tokens are contained within each Network Folder.
In addition, I added a cell containing the total value of tokens in each network folder, as well as a link to each network's block explorer (e.g. Etherscan).
I also got rid of the "import tokens" button, as the wallet should really import all tokens automatically, saving the users time.

Connected Sites

Allowing the users to disconnect their MetaMask wallet from an external site is a very important feature, as it serves as a way to avoid any security bridges. This important feature however, is completely hidden under a confusing three-dot menu button which contains the option to navigate to "Connected Sites". With this kind of nested navigation, it seems like it would be quite easy to forget to disconnect from any external sites and get potentially hacked by any external security bridge.
An always-visible menu to disconnect from external sites helps the user minimize risks.
My solution to this problem was to design and incorporate a "Connected Sites" section to the actual site, where it's always visible and therefore serves as a better reminder to the user to disconnect from any external sites.

User Accounts & Main Actions

MetaMask's main actions are all under the weird-looking profile picture at the top right corner.
These features are:
- User accounts
- Create account
- Connect to hardware wallet
- Support
- Settings
Once again, these could all come out of their nest, taking advantage of the extra space of the expanded window, and therefore being more accessible to the user.
Here's the result:
This neat left menu panel is always accessible. Before, these features were all hidden under an ugly and confusing user profile picture.

Receive Tokens

Whenever users wants to receive tokens from someone else, they have to either copy their address under their Account Name, or go to the, once again, confusing three-dot menu button > Account Details and copying it from there. This approach might be fine for most mid-to-advanced users, but shouldn't we strive to accomodate as many users as possible? Including our grandmas, yes. Say your grandma (or even you) wants to receive coins from someone else... the word receive seems to be the key word here. Why not keep it? That way your grandma (and you) can be fully happy and fulfilled.
This is why I chose to add a "Receive" button next to the other account actions (e.g. Buy, Send, etc.) This addition provides clarity and ease of use for the user.
Click on Receive anytime you want to Receive any coins ;)

Display Currency

After conducting some user testing, I received a ton of feedback on how convenient it would be to be able to change the display currency directly from the main page. Some users might prefer to see ETH/USD, some HEX/EUR.
My solution was to incorporate a simple yet effective currency-switch drop-down menu. This is what it looks like:
Simply click on the currency you want to change and you'll be able to pick any other from a drop-down list of currencies.

Overview

And last but not least, I added an Overview page, which esentially serves as a Home page. Here, the user will be able to see the total balance of all accounts combined, as well as their latest transactions.
